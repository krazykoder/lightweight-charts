## 2. Layer Invariants

The architecture is a strict five-layer stack. **No layer may import from a layer above it.**

```
Layer 1 — API        src/api/
Layer 2 — GUI        src/gui/
Layer 3 — Model      src/model/
Layer 4 — Views      src/views/
Layer 5 — Renderers  src/renderers/
```

### Dependency direction (enforced via TSC composite projects)

```
API → GUI → Model
API → Model
GUI → Views → Renderers
Model → (none of the above)
Renderers → (nothing in src/)
Formatters → (nothing in src/ except helpers)
Helpers → (nothing in src/)
```

- `src/helpers/` is a shared utility layer — any layer may import from it.
- `src/formatters/` may only be imported by `src/api/` and `src/model/`.
- Cross-layer imports that violate this order are **architectural violations (Class C)**.

## 3. Module Boundary Contracts

### 3.1 API Layer (`src/api/`)
- **Sole responsibility**: Translate user-facing calls into model mutations and return `IChartApi` / `ISeriesApi` / `IPriceScaleApi` / `ITimeScaleApi` interfaces.
- **Invariant**: All options accepted at the API boundary must be `DeepPartial<T>`. Merging with defaults happens in `src/api/options/`.
- **Invariant**: `DataLayer` owns all time-series normalization. No raw user data reaches the model directly.
- **Invariant**: No canvas or DOM manipulation in this layer.

### 3.2 GUI Layer (`src/gui/`)
- **Sole responsibility**: DOM creation, canvas sizing, DPR handling, mouse/touch event routing, and scheduling redraws via `requestAnimationFrame`.
- **Invariant**: `ChartWidget` is the only object that holds references to both the model (`ChartModel`) and all widgets (`PaneWidget`, `TimeAxisWidget`).
- **Invariant**: Redraws are always deferred through `InvalidateMask`. GUI must never call model methods that trigger immediate synchronous repaints.
- **Invariant**: Canvas pixel ratio handling is delegated entirely to `fancy-canvas` / `canvas-utils.ts`.

### 3.3 Model Layer (`src/model/`)
- **Sole responsibility**: Pure data and logic — coordinate transforms, data storage, price/time scale math. Zero DOM or canvas dependencies.
- **Invariant**: `ChartModel` is the root. All model state changes propagate upward via `InvalidateMask`, never via direct callback to GUI.
- **Invariant**: `PriceScale` and `TimeScale` are the only objects that perform coordinate-to-pixel conversions.
- **Invariant**: `PlotList` entries are always kept sorted by `TimePointIndex`. Inserting unsorted data is a contract violation.

### 3.4 Views Layer (`src/views/`)
- **Sole responsibility**: Bridge model state to renderer input data. Views are stateful caches — they must be invalidated when model state changes.
- **Invariant**: Views must not modify model state.
- **Invariant**: Pane views, price-axis views, and time-axis views are separate hierarchies and must not cross-reference each other.

### 3.5 Renderers Layer (`src/renderers/`)
- **Sole responsibility**: Stateless canvas drawing. Renderers receive pre-computed data from views and draw to `CanvasRenderingContext2D`.
- **Invariant**: Renderers must be stateless. No model or view references stored.
- **Invariant**: Renderers must not read from the DOM or trigger layout.

## 4. Data Flow Invariants

### 4.1 User Data → Internal PlotRow
```
User calls ISeriesApi.setData() / update()
  → ChartApi validates via data-validators.ts
  → DataLayer.setSeriesData() normalizes Time → TimePoint
  → DataLayer merges all series time points into unified TimeScale points
  → ChartModel receives SeriesChanges (added/updated/removed PlotRows)
  → Series.setData() populates PlotList (sorted by TimePointIndex)
  → InvalidateMask(Full) is created and handed to ChartWidget
```

### 4.2 Invalidation → Repaint
```
Model emits InvalidateMask via Delegate subscription
  → ChartWidget._invalidateMask is merged/accumulated
  → requestAnimationFrame schedules ChartWidget._drawImpl()
  → Each PaneWidget.paint(InvalidationLevel) is called
  → PaneWidget iterates pane views → calls renderer.draw()
```

### 4.3 Coordinate Transforms
```
Price value (number)
  → PriceScale.priceToCoordinate() → Coordinate (nominal number, pixels)

TimePointIndex (nominal number)
  → TimeScale.indexToCoordinate() → Coordinate
```

## 5. Nominal Types (Invariants)

These types are branded via `Nominal<T, Name>` and must never be mixed with plain `number`:

| Nominal Type | Base | Purpose |
|---|---|---|
| `Coordinate` | `number` | Pixel position on canvas |
| `TimePointIndex` | `number` | Logical position on time axis |
| `UTCTimestamp` | `number` | Unix timestamp in seconds |
| `BarPrice` | `number` | Single price value |

## 8. Out-of-Scope (Frozen)

The following are **not** part of this project and must not be introduced:

- Server-side rendering (SSR) — library asserts browser environment
- Multiple canvases per pane (one canvas per pane is the invariant)
- Direct WebGL or WebGPU rendering
- Framework-specific wrappers (React, Vue, etc.) inside `src/`
